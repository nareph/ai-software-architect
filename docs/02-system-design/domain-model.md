# Domain Model

## Approach

The domain model follows **Domain-Driven Design (DDD)** principles. The domain is organized around two main bounded contexts: **Project Management** and **Generation**.

---

## Core entities

### Project (Aggregate root)

The central entity. Encapsulates the user's initial description and all generated artifacts.

**Status transitions:**
```
draft → generating → completed
                  → partial → generating → completed
completed → generating (full regeneration)
* → archived
```

A `partial` project can move back to `generating` when the user relaunches failed artifacts, and reach `completed` if all artifacts are then successfully generated.

### Artifact

A document generated by AI, associated with a project.

**Types:** `business_analysis` | `architecture` | `database_schema` | `diagrams` | `backlog`

### ArtifactVersion

Each artifact modification creates a new version. Enables history and rollback.

### GenerationJob

Represents a complete execution of the generation pipeline for a project.

### GenerationStep

Represents an individual pipeline step (one step = one LLM call = one artifact type).

### Feedback

User feedback on an artifact, triggers targeted regeneration.

---

## Relations

```
User (1) ──────────── (N) Project
Project (1) ────────── (N) Artifact
Project (1) ────────── (N) GenerationJob
Artifact (1) ────────── (N) ArtifactVersion
Artifact (1) ────────── (N) Feedback
GenerationJob (1) ────── (N) GenerationStep
```

## Ubiquitous Language

| Term | Definition |
|---|---|
| **Project** | The central work unit — a business description + its generated artifacts |
| **Artifact** | A structured document produced by AI |
| **Version** | An immutable snapshot of an artifact at a given moment |
| **Generation** | The complete process of executing the LLM pipeline for a project |
| **Job** | An execution instance of the generation pipeline |
| **Step** | An individual LLM call producing one artifact |
| **Coherence** | Semantic alignment between artifacts of the same project |
| **Feedback** | A user instruction triggering a modification or regeneration |
| **Fallback** | Automatic switch to the secondary LLM (DeepSeek V4 Flash) |
| **Template** | A preconfigured project profile (SaaS, marketplace, etc.) |
